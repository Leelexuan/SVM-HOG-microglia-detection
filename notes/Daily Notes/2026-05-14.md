### Bug Fixes and Redundancy Removal

Full audit of the notebook. Issues found and fixed:

**Bugs (correctness):**

1. **Color space mismatch** ‚Äî `sliding_window_detect` was converting color on a float32 [0,1] image while `extract_features` converted on uint8 [0,255]. OpenCV `BGR2HSV` produces completely different value ranges per dtype (H=[0,360], S/V=[0,1] for float vs H=[0,179], S/V=[0,255] for uint8), so `color_hist`, `bin_spatial`, and `intensity_stats` were on mismatched scales at train vs inference time. Fixed by converting color on `img_uint8` in `sliding_window_detect`.

2. **`nblocks_win_c/f` hardcoded `cell_per_block=2`** ‚Äî formula `(win // ppc) - 1` is only correct when `cfg.hog_cell_per_block=2`. Corrected to general form `(win // ppc) - (cpb - 1)`.

3. **`evaluate_classifier` double-predicted** ‚Äî `clf.score(X, y)` internally calls `clf.predict` when `y_pred` already existed. Replaced with `(y_pred == y).mean()`.

**Dead code / redundancies:**

4. **`save_rotations`** ‚Äî never called, replaced long ago by `save_augmented_patches`. Deleted.
5. **Duplicate `import os`** ‚Äî appeared in both `cell-imports` and `cell-config`. Removed from Config cell.
6. **`max_noise_ratio` in Config** ‚Äî defined but never referenced anywhere. Removed.
7. **`cell-detection` placeholder** ‚Äî stale comment saying functions were moved. Deleted.
8. **`_clamp_crop` unused `end` parameter** ‚Äî passed by all callers but immediately overwritten by `start + window`. Removed the parameter; all callers updated.

**Documentation errors:**

9. **`build_inv_mask` docstring** ‚Äî said `cx, cy` but code reads `bbox["x"]`, `bbox["y"]`. Fixed.
10. **Duplicate `## 11` section header** ‚Äî `cell-md-threshold` and `cell-threshold` both labelled "## 11". Deleted the outdated one.
11. **Test run section** ‚Äî labelled "## 13" but HNM is already 13. Corrected to "## 14".
12. **`n_saved` print in HNM loop** ‚Äî said "total hard negatives in folder" but only counted saves within the current round. Changed to "new hard negatives this round".

### Kaggle Accelerator

This notebook uses only CPU-based libraries (scikit-learn, OpenCV, scikit-image, numpy). No GPU acceleration is possible. On Kaggle, select **None** as the accelerator (= CPU-only, 2 cores). GPU sessions would be entirely idle.

Also noted: `GridSearchCV` has `n_jobs=1` ‚Äî can be set to `n_jobs=-1` to use all available CPU cores and reduce grid search time.
