## Decision — `nohup` + `papermill`

- `tmux` not installed (needs sudo); `papermill` and `nbconvert` not installed.
- Chose **`nohup` + `papermill`** — zero sudo, fully detaches from the SSH session.
- `papermill` starts a headless kernel, executes every cell top-to-bottom, and writes
  outputs (including plots) into a *copy* notebook (`..._output.ipynb`), saving
  progressively after each cell — partial results survive a crash or mid-run check.
- It is fully decoupled from the VS Code notebook UI (separate kernel, separate file).
- **papermill 2.7 does not depend on `nbconvert`** — install `nbconvert` separately if
  HTML export of the output notebook is wanted.
- papermill has no cell-range option: it always runs the whole notebook. Subsetting
  requires editing the notebook (trimmed copy, blanked cells, or parameter flags).

## ⚠️ Discovery — the Run Is NOT Doing a Fresh Full Run

Investigation while monitoring the run:

- **papermill's log uses 1-based cell numbers.** Log "Cell 5" = notebook cell index 4
  (the `print(cfg)` config cell). Easy to misread as a runaway run.
- The pipeline training cells — preprocess (idx 10), `extract_raw_features` (idx 14),
  `GridSearchCV` (idx 16), `train_svm` (idx 18), model-save (idx 23) — produced
  **empty outputs** and completed near-instantly (cells 0–25 done in ~11 s).
- On-disk evidence confirms they did no work:
  - Patch folders `Train/Microglia`, `Train/Noise`, `Validate/*`, `Test/*` are all
    **empty (0 files)** — preprocessing never populated them.
  - `features_cache.npz` (16:10), `svm_clf.pkl` / `scaler.pkl` / `pca.pkl` (17:44) are
    **from yesterday** — not regenerated.
- The cold-start cell (idx 26) loaded yesterday's cached artifacts:
  `val images: 3, train samples: 14914 (including 0 hard negatives), val samples: 3036`.
- The HNM loop (idx 27) is now running detection on the 3 val images using
  **yesterday's model + features**, not freshly trained ones.

### First run (PID 23933) — FAILED

The run reached the §13 HNM loop and crashed:
`NameError: name 'fit_pipeline' is not defined`.

### Root cause
papermill **snapshots the input notebook once at launch**. The executed
`_output.ipynb` was structurally different from the current on-disk file — papermill
ran a stale, cold-start-configured variant in which the §5–§9 code cells were inactive.
So the §7 cell's `def fit_pipeline` (with `extract_raw_features`, `tune_svm`,
`train_svm`) never ran, and yesterday's cached artifacts were never overwritten.

`fit_pipeline` is a **function**, not a cached artifact. The cold-start cell loads
cached *data* (features, model `.pkl`s) but does not define the §7–§9 *functions* the
HNM loop needs. Each section cell mixes a function `def` with its slow execution call,
so the two cannot be skipped independently — the cold-start path is structurally
broken for the HNM loop.

Despite the crash, the run reached the HNM loop's `np.savez_compressed`:
- `features_cache.npz` rewritten ≈2.2 GB (round-1 hard negatives on the stale base).
- `HardNegatives/` repopulated with 15,701 crops, uniform 64×64 — the resize fix worked.
