# 2026-05-06

## Workflow Improvement Ideas (from AWS Learnings)

### 1. Treat Claude Context.md as a Steering Document

The AWS approach: steering docs are concise, living, team-reviewed references — not one-off notes.

**Apply to this project:**
- Keep `Claude Context.md` as the always-loaded steering doc (it already is)
- Split it into sections with clear inclusion intent:
  - **Always**: project overview, current status, key conventions
  - **Manual**: detailed error logs, old experiments, full pipeline descriptions
- Keep entries as concise bulleted points — fewer tokens, faster context load

---

### 2. Use a Session Prompt Template

AWS best practice: give the agent a role, clear context, and a specified output format before diving in.

**Apply at session start:**
```
Role: You are a machine learning engineer helping me improve a SVM-HOG microglia detection pipeline.
Context: [paste current status block from Claude Context.md]
Goal: [specific task for this session]
Constraints: [e.g. no data leakage, detection-level metrics only]
Output format: [notebook cells / explanation / updated notes]
```

This replaces ad-hoc task descriptions with a reproducible structure.

---

### 3. Add Agent Hooks to the Workflow

AWS examples: lint on change, prompt history, test coverage, security scanner.

**Relevant hooks for this project:**
- **End-of-session hook**: auto-trigger `/update-context` when Claude Code stops — ensures daily notes are never skipped
- **Prompt history log**: log every session prompt to a `logs/` file for a searchable audit trail of how the model was used across sessions
- **Notebook guard**: when a new `.ipynb` cell is added that trains a model, prompt to check if train/test split happens *before* augmentation (data leakage guard)

---

### 4. Structure Session Briefs Like a Sprint

AWS Appforce uses Product Owner + Tech Lead roles across a full sprint cycle.

**Apply a lightweight version:**
At session start, generate a mini-brief:
- **Current State**: what was done last session
- **Goal**: what to accomplish today
- **Blockers**: known issues (e.g. patch metrics ≠ detection quality)
- **Definition of Done**: what "finished" looks like (e.g. detection-level precision/recall curve plotted)

This maps cleanly onto the existing daily notes format — just make it more structured.

---

### 5. Agentic Data Priorities → Training Data Hygiene

AWS framework: agent quality = data quality + discoverability + access.

**Apply to training pipeline:**
- **Quality**: fix the data leakage issue (split before augmentation) — directly improves model reliability
- **Discoverability**: document what each CSV (`Image_ROIs*.csv`) contains at the top of `Claude Context.md`
- **Audit**: log which images were used for train vs. test in each experiment run (currently implicit in notebook cells)

---

### 6. Steering Document Lifecycle

AWS process: Vibe → Refine → Review → Present → Merge → Iterate.

**Apply to `Claude Context.md`:**
1. After each meaningful session, run `/update-context` (already doing this)
2. Periodically review and prune stale sections (old experiment notes)
3. Before starting a new model approach, re-read the "Recommended Next Steps" section — treat it like a backlog

## Interview Prep Plan (2026-05-06)

- If YOLOv8 underperforms SVM at this data scale, that's the talking point: classical HOG+SVM still wins with small datasets
- `Experiment Story.md` created — scaffold for filling in experiment details before touching slides
