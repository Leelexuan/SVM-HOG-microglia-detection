# 2026-05-08

## Pipeline Refactor — Train / Validate / Test + HNM

### Split structure finalised
- **Train / Validate**: split from `Source_images/` at the image level (`val_size=0.2` → ~10 train, ~3 val). Augmentation applied to train only — no leakage.
- **Test**: fixed 2 images in `test_dir/` — never touched until final evaluation. Patches extracted once into `Processed_training_images/Test/` by `preprocess_all_images`.
- Validate is the old "test" set. Test is the new held-out set.

### Feature extraction redesign
Separated into two functions to make HNM efficient:
- `extract_raw_features(cfg)` — slow, called once. Returns `X_train_raw`, `X_val_raw`, `y_train_raw`, `y_val_raw`.
- `fit_pipeline(X_train_raw, X_val_raw, ...)` — fast, called each HNM round. Refits scaler + PCA, returns `X_train`, `X_val`, `y_train`, `y_val`, `scaler`, `pca`.
- Raw features saved to `features_cache.npz` (path in `cfg.features_cache`) after extraction and after each HNM round — survives kernel restarts.

### Hard Negative Mining loop
- Dedicated folder: `cfg.hnm_folder = Processed_training_images/HardNegatives/` — accumulates crops across all rounds, never cleared.
- Each round runs detection on validate images with `cfg.detection_threshold = 0.0` (raw SVM boundary — no threshold during mining).
- Only new crops (tracked via `n_hn_extracted` counter) have features extracted — existing crops are not re-processed.
- After each round: append new features to `X_train_raw`/`y_train_raw` → save cache → refit pipeline → retrain SVM.
- Converges when `new_hn_paths` is empty.

### Threshold tuning (after HNM)
- `tune_detection_threshold(val_paths, ...)` — runs full detection on validate images at multiple thresholds, picks the one maximising detection-level F1 (IoU ≥ 0.5 matched boxes).
- Sweep range derived from validate patch score distribution (`pos_scores.min()` → `pos_scores.max()`), not hardcoded.
- Patch-level F1 threshold tuning removed — wrong signal for detection.

### Final evaluation (once, at the end)
- Patch-level: extract test patch features using final `scaler` + `pca`, call `evaluate_classifier`.
- Detection-level: sliding window on `test_dir/` images, compare boxes vs GT with `evaluate_detections` (IoU ≥ 0.5).

### Key decisions
- No threshold during HNM: lower threshold = more FPs collected per round = faster convergence.
- No re-tuning threshold each HNM round: only tune once on the final converged model.
- HNM mines from validate images only — test images never used for model improvement.
- Naming convention: `_raw` suffix for unscaled arrays (`X_train_raw`, `y_train_raw`), no suffix for PCA-reduced arrays ready for SVM.
