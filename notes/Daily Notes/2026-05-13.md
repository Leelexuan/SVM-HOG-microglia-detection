### Naming Discrepancies Identified

- `fit_pipeline` returns `y_train_raw` / `y_val_raw` unchanged but the call site unpacks them as `y_train` / `y_val`, misleadingly implying they were transformed (unlike the `X_` arrays which are genuinely scaled + PCA-reduced).
- Two separate held-out splits coexist with inconsistent naming: `X_val`/`y_val` (validation, used during training) vs `X_test`/`y_test` (final held-out, introduced later in the notebook).

### Cold-Start Refactor for HNM Loop

Goal: allow the HNM loop and everything after it to run on a kernel restart without re-running preprocessing or feature extraction.

**Changes made to `SVM_original_refactored.ipynb`:**

1. **`cell-imports`** — added `import joblib` to top-level imports.

2. **`cell-config`** — added `val_paths_cache: str` to Config so the path is centralised.

3. **`cell-preprocess`** — saves `val_paths` to `val_paths.txt` after `preprocess_all_images()` returns. Previously this list was only held in memory and lost on restart.

4. **New cold-start cell** (inserted before HNM loop) — loads `val_paths.txt`, `features_cache.npz`, and the three `.pkl` files; reconstructs `X_train`/`X_val`/`y_train`/`y_val` by transforming with loaded scaler/pca (no refit); sets `n_hn_extracted` from existing HNM folder count.

5. **`tune_detection_threshold`** — fixed global variable dependency: `X_val` and `y_val` are now explicit parameters. Call site updated accordingly.

6. **HNM loop** — added `joblib.dump` calls at the end of each round so saved artifacts stay in sync with the latest `X_train_raw` state (previously only saved once before the loop).

7. **Deleted** the stale commented-out reload stub.
